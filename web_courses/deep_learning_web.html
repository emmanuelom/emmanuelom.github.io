Sección 01

¿Qué es el Deep Learning?

El deep learning es una rama del Machine Learning que usa redes neuronales con múltiples capas para aprender representaciones jerárquicas de los datos — sin features diseñadas a mano.

📦
Aprende de datos crudos
Imágenes, texto, audio — sin necesitar ingeniería manual de features. La red descubre qué importa.
🏗️
Capas profundas
Cada capa extrae representaciones más abstractas: bordes → texturas → formas → objetos.
🧬
Inspiración biológica
Modela (libremente) las conexiones sinápticas del cerebro. Las neuronas se activan según el estímulo.

Sección 02

La neurona artificial

Una neurona recibe entradas, las pondera con pesos, suma un sesgo y aplica una función de activación. Arrastra los sliders y observa cómo cambia la salida en tiempo real.

Función de activación:

z = w₁x₁ + w₂x₂ + b
—
ŷ = f(z)
—
neurona.py
import numpy as np

def sigmoid(z): return 1 / (1 + np.exp(-z))
def relu(z):    return np.maximum(0, z)

def neurona(x, w, b, f=sigmoid):
    """
    x : vector de entradas  (n,)
    w : vector de pesos     (n,)
    b : sesgo escalar
    f : función de activación
    """
    z = np.dot(w, x) + b    # suma ponderada
    return z, f(z)           # (pre-activación, salida)

# Ejemplo
x = np.array([1.0, 0.5])
w = np.array([0.8, -0.5])
z, y_hat = neurona(x, w, b=0.2)
print(f"z={z:.3f}  ŷ={y_hat:.3f}")
Sección 03

Red neuronal multicapa (MLP)

Un MLP apila capas de neuronas. La información fluye del input al output en el forward pass. Haz clic en las neuronas para ver sus conexiones activarse.

Notación matricial
h = f(W¹x + b¹)
ŷ = g(W²h + b²)
Teorema de Aproximación Universal
Una red con al menos una capa oculta suficientemente grande puede aproximar cualquier función continua. Más capas = jerarquías más eficientes.
Hiperparámetros clave
📐 Profundidad — número de capas
↔️ Ancho — neuronas por capa
⚡ Activación — ReLU, Sigmoid…
🎯 Loss — MSE, CrossEntropy
Sección 04

Introducción a PyTorch

PyTorch usa tensores (como arrays NumPy) y calcula gradientes automáticamente con autograd. Así podemos entrenar redes con unas pocas líneas.

tensores.py
import torch

# Crear tensores
x = torch.tensor([1.0, 2.0, 3.0])
A = torch.randn(3, 4)          # matriz 3×4 aleatoria
b = torch.zeros(4)             # vector de ceros

# Operaciones (igual que NumPy)
y = A.T @ x + b                # shape: (4,)

# ── Autograd ────────────────────────────────
x = torch.tensor([2.0], requires_grad=True)
loss = x**2 + 3*x + 1
loss.backward()
print(x.grad)   # tensor([7.])  ← d/dx = 2x+3 en x=2
modelo.py
import torch.nn as nn

class MiRed(nn.Module):
    def __init__(self):
        super().__init__()
        self.red = nn.Sequential(
            nn.Linear(3, 64),    # 3 entradas → 64 neuronas
            nn.ReLU(),
            nn.Linear(64, 32),   # 64 → 32
            nn.ReLU(),
            nn.Linear(32, 1)    # 32 → 1 salida
        )

    def forward(self, x):
        return self.red(x)

modelo = MiRed()
print(sum(p.numel() for p in modelo.parameters()))  # 2369 params

El ciclo de entrenamiento

Cada epoch: forward → calcular pérdida → backprop → actualizar pesos. Observa el descenso del gradiente en vivo.

training_loop.py
criterio    = nn.BCEWithLogitsLoss()
optimizador = torch.optim.Adam(modelo.parameters(), lr=1e-3)

for epoch in range(200):
    # 1. Forward pass
    y_pred = modelo(X_train)

    # 2. Calcular pérdida
    loss = criterio(y_pred, y_train)

    # 3. Limpiar gradientes
    optimizador.zero_grad()

    # 4. Backpropagation
    loss.backward()

    # 5. Actualizar pesos  w ← w − lr·∇w
    optimizador.step()

    if epoch % 20 == 0:
        print(f"Epoch {epoch}  loss={loss.item():.4f}")
